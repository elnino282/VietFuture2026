# Internationalization (i18n) Guide

This document explains how to use and maintain the internationalization system in the ACM Web Platform.

## Overview

The project uses **i18next** with **react-i18next** for internationalization, supporting:
- **English (en-US)** - Default language
- **Vietnamese (vi-VN)**

## Quick Start

### Using translations in components

```tsx
import { useI18n } from '@/hooks/useI18n';

function MyComponent() {
    const { t, locale, setLocale } = useI18n();
    
    return (
        <div>
            <h1>{t('common.appName')}</h1>
            <p>{t('auth.signIn.title')}</p>
            <button onClick={() => setLocale('vi-VN')}>
                Switch to Vietnamese
            </button>
        </div>
    );
}
```

### Using formatters

```tsx
import { useFormatters } from '@/hooks/useI18n';

function PriceDisplay({ amount }: { amount: number }) {
    const { formatCurrency, formatDate, formatWeight } = useFormatters();
    
    return (
        <div>
            <p>Price: {formatCurrency(amount, 'VND')}</p>
            <p>Date: {formatDate(new Date())}</p>
            <p>Weight: {formatWeight(50, 'KG')}</p>
        </div>
    );
}
```

## Translation Key Naming Conventions

### Structure
Use **dot notation** with logical namespacing:

```
{namespace}.{feature/page}.{element}
```

### Namespaces

| Namespace | Description | Example |
|-----------|-------------|---------|
| `common` | Shared UI text | `common.save`, `common.cancel` |
| `nav` | Navigation items | `nav.dashboard`, `nav.settings` |
| `userMenu` | User menu items | `userMenu.profile`, `userMenu.signOut` |
| `auth` | Authentication pages | `auth.signIn.title`, `auth.signUp.email` |
| `preferences` | Preferences page | `preferences.unitSettings.title` |
| `toast` | Toast notifications | `toast.success`, `toast.preferencesUpdated` |
| `validation` | Form validation | `validation.required`, `validation.invalidEmail` |
| `errors` | Error messages | `errors.network`, `errors.unauthorized` |
| `dashboard` | Dashboard | `dashboard.welcome`, `dashboard.overview` |
| `table` | Table components | `table.noResults`, `table.rowsPerPage` |

### Key Naming Rules

1. **Use camelCase** for all keys
2. **Be descriptive** but concise
3. **Group related keys** under the same parent
4. **Use action verbs** for buttons: `save`, `cancel`, `submit`

## Adding New Translations

### Step 1: Add to both JSON files

**en.json:**
```json
{
    "myFeature": {
        "title": "My Feature",
        "description": "This is a description",
        "saveButton": "Save Changes"
    }
}
```

**vi.json:**
```json
{
    "myFeature": {
        "title": "Tính năng của tôi",
        "description": "Đây là mô tả",
        "saveButton": "Lưu thay đổi"
    }
}
```

### Step 2: Use in component

```tsx
const { t } = useI18n();

<h1>{t('myFeature.title')}</h1>
<p>{t('myFeature.description')}</p>
<Button>{t('myFeature.saveButton')}</Button>
```

## Placeholders (Interpolation)

Use `{{variable}}` syntax for dynamic values:

**JSON:**
```json
{
    "greeting": "Hello, {{name}}!",
    "itemCount": "You have {{count}} items"
}
```

**Usage:**
```tsx
t('greeting', { name: 'John' }) // "Hello, John!"
t('itemCount', { count: 5 })    // "You have 5 items"
```

## Do NOT Translate List

Keep these terms unchanged in ALL languages:

- **Framework/Library names:** React, Next.js, Spring Boot, TailwindCSS, Radix UI, TanStack Query
- **Technical terms:** API, OAuth, JWT, WebSocket
- **Product names:** Google, Firebase, Cloudinary, Stripe
- **Abbreviations:** URL, ID, PDF, CSV

## Vietnamese Translation Guidelines

### Tone
- Use formal, professional language
- Avoid literal translations that sound unnatural
- Keep UI text concise

### Common Patterns

| English | Vietnamese | Notes |
|---------|------------|-------|
| Sign in | Đăng nhập | |
| Sign out | Đăng xuất | |
| Sign up | Đăng ký | |
| Create Account | Tạo tài khoản | |
| Preferences | Tùy chọn | Or "Cài đặt" |
| Settings | Cài đặt | |
| Theme | Giao diện | |
| Language | Ngôn ngữ | |
| Profile | Hồ sơ | |
| Save | Lưu | |
| Cancel | Hủy | |
| Delete | Xóa | |
| Edit | Chỉnh sửa | |
| Loading... | Đang tải... | |

### Avoid

- ❌ "Đăng ký" for "Register" (use "Đăng ký" only for Sign up context)
- ❌ Literal translations of idioms
- ❌ Inconsistent terminology across pages

## Language Detection Priority

1. **User preference from API** (locale in user preferences)
2. **localStorage value** (`acm_language`)
3. **Browser language** (navigator.language)
4. **Default fallback:** `en-US`

## Testing Checklist

- [ ] Switch language from user menu updates UI immediately
- [ ] Preferences locale dropdown reflects current locale
- [ ] Saving preferences persists and survives refresh
- [ ] No "flash" of wrong language on reload
- [ ] No broken layouts due to longer Vietnamese strings
- [ ] All visible text is translated
- [ ] Placeholders work correctly
- [ ] Date/number/currency formatting respects locale

## Troubleshooting

### Translations not showing

1. Check if the key exists in both JSON files
2. Verify the key path is correct (no typos)
3. Check browser console for i18n warnings

### Language not persisting

1. Verify localStorage is not blocked
2. Check if `LANGUAGE_STORAGE_KEY` is consistent
3. Ensure `changeLanguage` is called correctly

### Layout issues with Vietnamese

Vietnamese text can be longer than English. Solutions:
- Use `truncate` class for overflow
- Set `min-width` on containers
- Use responsive text sizing
